# ЁЯУЬ Change Log тАФ Version v1.1.0
**Release Date:** May 19, 2026  
**System Architecture:** Room DB v2 Migrations, Kotlin, and Material Design UI Event Observers.

This release introduces the highly anticipated **Multi-Type Loan System**, transforming the application from a simple fixed-rate tracker into a versatile debt portfolio manager capable of tracking fixed rates, reducing balances, interest-only payments, and Islamic Sharia-compliant financing.

---

## ЁЯЪА New Features & Enhancements

### 1. ЁЯзо Automatic Loan Principal Calculation
- **On-The-Fly Real-Time Calculation**: Programmed real-time observers that automatically calculate the original loan principal by multiplying the monthly payment amount by the number of monthly schedules (`monthlyAmount * paymentMonths`).
- **Interactive UI Event Triggers**: Attached a dynamic `TextWatcher` to `editAmount` and integrated callback calculations inside the **Start Month/Year** (`cardStartDate`) and **End Month/Year** (`cardEndDate`) wheel pickers. Modifying any of these values recalculates and displays the principal immediately.
- **Form Verification & Safety**: Upgraded `activity_create_edit_debt.xml` to render the Principal input field (`editPrincipal`) in a non-editable, read-only mode (`android:focusable="false"`, `android:cursorVisible="false"`, and `android:clickable="false"`). This prevents accidental or incorrect manual entries, ensuring absolute mathematical accuracy across the database.
- **Edit Mode Restoration Hook**: Added dynamic recalculation hooks to `loadDebtDetails()` in `CreateEditDebtActivity.kt` so that existing obligations correctly display their calculated values on launch.

### 2. ЁЯПж Multi-Type Loan Support (Fixed, Reducing, Islamic)
- **Math Calculation Engine (`LoanCalculator`)**: Created a dedicated calculations utility to calculate next due payments, amortized schedules, and remaining principal balances:
  - **Fixed-Rate Installment**: Standard annuity calculations with fixed monthly payments.
  - **Reducing Balance / Credit Lines**: Computes daily compounding interest on the active outstanding balance. Suggests a minimum monthly payment equal to the greater of `(Interest + 1% Principal)` or `(MinPaymentPercent * Balance)`.
  - **Interest-Only (Bullet/Balloon)**: Calculates monthly interest-only payments (`Principal * Rate / 12`), with the full principal maturing in a lump sum at the end date.
  - **Islamic Financing (Murabaha/Ijarah)**: Uses a flat profit rate and tenor length to calculate total financed expected profit.
- **Enums & Custom Type Converters**: Declared `InterestType` and `PaymentType` enums and registered database `Converters` to automatically map string enums into SQLite records.
- **Room Migration (v1 -> v2)**: Incremented the database version to `2` and created a safe migration script (`MIGRATION_1_2`) that executes `ALTER TABLE debts ADD COLUMN...` statements, upgrading all existing user records safely without data loss.

### 3. ЁЯОи Dynamic & Responsive Form Layouts
- **Outlined Material Exposed Dropdowns**: Integrated premium dropdowns for **Interest Type** and **Payment Type** with outrageously clean ripple boundaries and locked drop-down overlays to prevent OEM dark-mode overrides.
- **Visibility & Form Managers**: Toggle layout controls dynamically:
  - Selecting **Reducing Balance** displays additional Outlined inputs for **Credit Limit** and **Minimum Payment (%)**.
  - Shifts label hints contextually (e.g., changes "Interest Rate (%)" to "Profit Rate (%)" when **Islamic Finance** is selected).
- **Edit Mode Restoration**: Pre-populates all inputs, toggles field visibilities, and binds dropdown item labels accurately based on the active record details.

### 4. ЁЯУК Adaptive Analytics & Detail Screen
- **Expanded Information Grid**: Renders comprehensive loan metadata: **Loan Type**, **Interest/Profit Rate**, **Credit Limit**, and **Remaining Balance**.
- **Dynamic Field Visibility**: Displays information conditionally:
  - Outstanding balances and credit limits show only when applicable (reducing balance/non-zero).
  - Dynamically changes titles (e.g. shifts "Monthly Amount" to "Suggested Payment" for reducing balance credit lines).
- **Automated Bottom-Sheet Payments**: When logging a payment, the modal dynamically pre-fills using the calculator's computed suggested payment instead of a static raw value.

### 5. ЁЯУВ Upgraded CSV Backup & Restore
- **Expanded Serializer**: Upgraded `CsvBackupManager` to export all seventeen (17) columns tracking interest types, credit limits, and balances.
- **100% Backward-Compatibility**: Refactored the CSV import parser. It successfully processes both old 10-column backups (falling back to standard fixed installments) and new 17-column backups with absolute safety.
