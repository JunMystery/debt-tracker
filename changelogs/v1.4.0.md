# Debt Tracker - Release Changelog v1.4.0

This release marks the beginning of the v1.4.0 cycle. All subsequent modifications, new features, and layout polishing will be tracked here.

## 🚀 Features & Enhancements

### 1. ⚙️ Settings UI Polish & Title Visibility
- **Enlarged Collapsible Headers**: Increased the text size of collapsible accordion section titles (`General`, `Notifications`, `Backup`, `Import/Restore`) from `13sp` to `16sp` to improve hierarchy, readability, and contrast against internal options.

### 2. 📱 Swipe-to-Reveal Delete Action & Long-Press Option Dialog
- **Swipe-to-Reveal Action**: Replaced the static three-dots overflow menu button on the debt cards with a clean swipe-left gesture. Swiping left translates the card foreground to reveal a hidden, red action background container with a white trash can icon (`ic_delete.xml`).
- **Interactive Action & Touch Control**: Implemented a custom touch feedback class `SwipeRevealTouchListener` that snaps the card open to `-80dp` on swipe release and snaps it closed on click. Swiping is disabled on the Completed obligations list view.
- **Hold-to-Option Dialog**: Implemented `OptionsDialog.kt` to show a Material 3 choice dialog (Edit, Delete) upon long-pressing a debt card, replacing the direct deletion trigger.
- **Upgraded OptionsDialog UI**: Replaced the default items list dialog with a custom styled [dialog_options.xml](file:///e:/Github/Debt-Tracker/app/src/main/res/layout/dialog_options.xml) layout featuring a clean card container, rounded corners, and three distinctive action buttons arranged horizontally with uniform width:
  - **Edit Button**: White text on Blue background (`@color/dt_primary`).
  - **Delete Button**: White text on Red background (`@color/dt_error`).
  - **Cancel Button**: Red text on White background with a solid red border.
  - **Short Translations**: Added `@string/edit_short` ("Sửa" / "Edit") to keep the button sizes perfectly uniform and compact across all localizations.

## 🔧 Bug Fixes
- **Remaining Months Calculation Correction**: Fixed an off-by-one counting error where the remaining months calculation counted months prior to the debt's start month (e.g. counting 7 months instead of 6 months when the current month is May, but the debt starts in June and ends in November). The calculation now bounds the starting point to the debt's `startYearMonth` if the current month is before the debt start date.
- **Java Language Server & Gradle JDK 21 Fallback Fix**: Fixed compilation errors by redirecting the build tools from the early-access JDK 26 (which is incompatible with Android 35's `JdkImageTransform` execution) and VS Code's stripped-down extension JRE (which lacks developer binaries like `jlink.exe`). Configured the project's `gradle.properties`, `.vscode/settings.json`, and `gradle-daemon-jvm.properties` to target the full JDK 21 installation located at `C:\Users\Jun\.gradle\jdks\eclipse_adoptium-21-amd64-windows.2`, ensuring compilation tasks like `compileDebugJavaWithJavac` run with a standard compatible toolchain.
- **Swipe-to-Reveal Interaction & Touch Propagation Fixes**: 
  - Fixed bugs in the swipe gesture where users could not swipe the card back (to the right) to hide the delete action.
  - Resolved touch propagation issues where touching another card while one was swiped open erroneously triggered click/dialog events on the target.
  - Implemented `dispatchTouchEvent` in activities to capture all clicks outside the open card. Any touch outside (whether on empty spaces, other cards, toolbars, or buttons) is consumed to immediately deactivate/close the open swipe without triggering accidental actions.
  - **Long-Press Dialog Interruption**: Fixed a bug where a swipe gesture still triggered the `OptionsDialog` when held down. Added `view.cancelLongPress()` inside `SwipeRevealTouchListener` to cancel pending long presses once dragging starts.
  - **Eliminated Dialog Dismissal Delay**: Disabled fade transitions of `OptionsDialog` using `dialog.window?.setWindowAnimations(0)` so that window focus is returned immediately to the main screen, allowing users to instantly select/hold other card items.
  - **Premium Holding Micro-Animations**: Enhanced the card touch feedback by configuring a semi-transparent blue ripple (`app:rippleColor="#303F51B5"`) on the foreground layout and a tactile scale-down animation (`0.97f` scale) during active presses, returning instantly to `1.0f` scale upon finger release, swipe init, or touch cancel.
  - **Immediate Post-Dialog Hold/Select Fix**: Resolved a bug where dismissing the `OptionsDialog` prevented immediate holds or clicks on other cards (requiring a release and second attempt). This occurred because `openedSwipeListener` tracked the last touched card even if it wasn't swiped open, triggering a swipe close callback that consumed the first `ACTION_DOWN` event. Updated `onSwipeStateChanged` inside both `UpcomingAdapter` and `DebtListAdapter` to only return `true` (and consume touch) if the tracked card is actively open (`isOpened() == true`).
