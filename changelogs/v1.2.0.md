# 📜 Change Log — Version v1.2.0

**Release Date:** May 19, 2026  
**System Architecture:** Room DB v3 Migrations, Standalone Preferred Currency Picker, 100% Offline Static Currency Conversions, and Real-Time Numeric Custom Format Watchers.

This release introduces a **Standalone Preferred Currency System** and resolves a set of core stability vulnerabilities. By severing the link between system locale and active currency selection, users can now select any currency display preference manually in settings (independent of app language choice). Furthermore, the automatic real-time exchange rate API fetching has been completely stripped in favor of a **100% offline, zero-network** static engine to achieve maximum stability, data integrity, and privacy.

---

## 🚀 New Features & Enhancements

### 1. 🎛️ Standalone Preferred Currency Setting (Independent of Language)
- **Settings Card Integration**: Inserted a premium dropdown selector card for **Preferred Currency** in settings, styled exactly to match the dark/glassmorphic look of the existing options.
- **Independent SharedPreferences Storage**: Persists the user's manual selection (`USD`, `VND`, `GBP`, `CNY`, `JPY`, `KRW`, `TWD`) under SharedPreferences `pref_preferred_currency`.
- **Locale-Independent Formatting**: Enhanced `CurrencyUtils.kt` to format monetary sums directly inside the manually selected preferred currency, independent of the current system language.
- **Immediate Dashboard Refresh**: Engineered a custom LiveData `refreshTrigger` in `DashboardController` coupled with `onResume()` hooks in `MainActivity`. Changing preferred settings instantly updates all dashboard aggregations, lists, cards, and views upon returning to the home screen.

### 2. 📴 100% Offline Static Currency Conversions (API Removed)
- **Automatic Exchange Rates API Deletion**: Completely removed the real-time HTTP networking coroutines, cache maps, and web-fetching functions. The application no longer performs any internet calls or network tracking.
- **Static High-Fidelity Conversion Map**: Retained a lightweight, memory-efficient offline rates helper inside `ExchangeRateService.kt` to perform instant offline conversions when summing multi-currency obligations (e.g. `USD = 1.0`, `VND = 25400.0`).
- **Smooth Dashboard Aggregation**: Multi-currency obligations (USD, VND, etc.) are converted to the manually selected preferred currency on-the-fly, keeping the math 100% correct, synchronous, and offline.

### 3. ⌨️ Custom Real-Time Currency Soft Input Watcher
- **Soft Keyboard Number Enforcement**: Attached a programmatic keyboard class hook (`InputType.TYPE_CLASS_NUMBER`) coupled with digit key listener restrictions (`DigitsKeyListener.getInstance("0123456789")`). Tapping on currency input fields now **strictly opens the native numeric keyboard** (blocking standard character/symbol keyboards completely).
- **VND Thousand Dot Separator**: Created custom regex processors that automatically insert dots after every three digits from the right, appending the `" VND"` suffix (e.g. user types `1000000` ➔ dynamically formats to `1.000.000 VND`).
- **USD Fractional Cents-to-Dollars**: Formats automatically from cents as the user types, placing exactly two decimal numbers before the `" USD"` suffix (e.g. typing `100` ➔ `1.00 USD`; typing `12345` ➔ `123.45 USD`).
- **Standard Localized Fallbacks**: Applies appropriate fractional rules for GBP, CNY, EUR, TWD (start from cents) and non-fractional formatting rules for JPY, KRW (integers with thousand separators).
- **Presentation-Persistence Separation**: The custom watcher handles all visual representations on-the-fly, while exposing a clean Kotlin trailing lambda callback `onRawValueChanged: (Double) -> Unit` to supply raw, unformatted decimal values directly to the host Activity.

### 4. 🏦 Room Database Schema Migration (v2 ➔ v3)
- **Currency Code Parameterization**: Updated the `Debt` data model to include `val currencyCode: String = "USD"`, tracking the exact currency tag in which each loan obligation was recorded.
- **Safe SQLite Schema Migration**: Configured a robust Room migration script `MIGRATION_2_3` that performs standard SQLite alteration tasks safely, defaulting existing columns to `'USD'` to guarantee **no data loss**:
  ```sql
  ALTER TABLE debts ADD COLUMN currencyCode TEXT NOT NULL DEFAULT 'USD'
  ```
- **Builder Integration**: Registered the new migration inside `AppDatabase.getInstance()`'s build chain.

### 5. 🧯 Crash-Proof Date Formatting Fallbacks
- **OS Template Protection**: Wrapped date formatting hooks inside `DateUtils.kt` in robust `try-catch` blocks. If an OEM system or a locale return formatting skeletons with unescaped literal letters (such as `"tháng"` in Vietnamese) that crash `DateTimeFormatter`, the methods catch the exception and fall back to bulletproof standard patterns (`"dd/MM/yyyy"` and `"MM/yyyy"`). This prevents all list-rendering and activity-recreation crashes.
- **Strict Locale Date Serialization**: Forced `Locale.US` within all date-to-database string serializers inside `CreateEditDebtActivity.kt` (`String.format(Locale.US, "%04d-%02d", ...)`) to ensure YYYY-MM dates are strictly standard ASCII digits across all devices.

### 6. 📂 CSV Backup & Restore Parity
- **Serialized Column Expansion**: Expanded `CsvBackupManager` to serialize `currencyCode` as the 18th column in standard backups.
- **Perfect Backward-Compatibility**: Refactored the CSV parser to successfully import both older 10/17-column files (falling back to `"USD"`) and new 18-column formats without errors.
