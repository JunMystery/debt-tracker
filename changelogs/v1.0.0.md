# 📜 Change Log — Version v1.0.0
**Release Date:** May 19, 2026  
**System Architecture:** Kotlin, Room (SQLite) with KSP, MVC/Repository Pattern, AlarmManager, MediaStore, and Jetpack.

This is the inaugural change log for **Simple Debt Tracker** (v1.0.0), detailing every single feature, architectural optimization, custom widget, translation expansion, visual polish, and performance improvement implemented from the beginning of this development session.

---

## 🚀 Key Features & Functional Upgrades

### 1. ⚙️ Redesigned Settings Screen & App Customization
- **Access Gear Toolbar Icon**: Replaced standard switch text with a clean, vector‑based Settings Gear button (`btnSettings`) on the top‑right of the Main Dashboard (`activity_main.xml`), utilizing the custom vector asset `ic_settings.xml`.
- **Exposed Dropdown Customization**: Created a dedicated `SettingsActivity` using modern Material Design exposed dropdown widgets (`AutoCompleteTextView` inside TextInputLayout).
- **Flag-Prefixed Language Picker**: Integrated flag emojis to make locale identification visually striking:
  - 🇺🇸 English (US) — tag `en-US`
  - 🇬🇧 English (UK) — tag `en-GB`
  - 🇻🇳 Tiếng Việt — tag `vi-VN`
  - 🇨🇳 简体中文 — tag `zh-CN`
  - 🇨🇳 繁體中文 — tag `zh-TW` (dynamically configured with the China `🇨🇳` flag)
  - 🇯🇵 日本語 — tag `ja-JP`
  - 🇰🇷 한국어 — tag `ko-KR`
- **Modern Jetpack Locale API Integration**: Leveraged `AppCompatDelegate.setApplicationLocales` for seamless locale switching without complex context wrapper hacks.
- **State-Restoration Language Fix**: Added `android:saveEnabled="false"` to exposed dropdown menus in the settings layout. This stops the view hierarchy from preserving outdated literal translations of dropdowns when recreated, forcing the layout engine to render fully updated translated items immediately.

### 2. 💱 Multi-Currency Formatting & Real-Time Exchange Rate Ticker
- **Dynamic Multi-Currency Adapter**: Upgraded `CurrencyUtils.kt` to dynamically read locale tags and automatically format outputs matching local currency symbols (`$`, `£`, `₫`, `¥`, `NT$`, `₩`).
- **Fractionless Rounding**: Configured JPY, KRW, VND, and TWD to automatically hide decimal fractions, while preserving standard cents for USD and GBP.
- **Dynamic Exchange Service (`ExchangeRateService.kt`)**: Integrates real-time exchange rates from Budjet API (`GET https://api.budjet.org/fiat/USD/{target_currency}`) with an active multi-key cache (1-hour expiry).
- **Dashboard Conversion Ticker**: Appended a real-time conversion sub-card (`textConvertedDue`) beneath the monthly stats. It dynamically displays conversions:
  - USD converts to VND.
  - All other currencies (GBP, VND, CNY, TWD, JPY, KRW) convert to USD alongside the exact exchange rate (e.g. `≈ $120.50 (Rate: 7.24 ¥/USD)`).

### 3. ⏰ Dynamic Equidistant Notification Scheduling Engine (AlarmManager)
- **Equidistant Spacing Model (Interval-Free)**: Replaced complex frequency/interval dropdown menus with a clean equidistant spacing algorithm:
  - *Frequency = 1*: Schedules a single reminder at the start hour.
  - *Frequency > 1*: Automatically divides the active daily time window (Start Hour to End Hour) into perfectly equidistant moments, removing duplicates and complex intervals.
  - Cleaned up all layout references and purged redundant interval tags inside `strings.xml`.
- **Double-Wheel Time Selector (HH:MM)**: Upgraded hour pickers in `WheelPickerDialog` to display two side-by-side scrolling wheels: Hours (running fully from `0` to `23` for complete day coverage) and Minutes (running `0` to `59`), housed inside custom theme overlays.
- **Timetable Preview Dialog**: Added a clickable info icon (`ibPreviewInfo`) next to the frequency settings. Tapping it shows a Material dialogue containing a comprehensive timeline list of all daily notification times (e.g., `09:00, 13:00, 17:00, 21:00`).
- **Resilient Exact Alarm Pipeline**:
  - `DebtReminderScheduler`: Optimizes performance and complies with Android exact alarm system limits by computing and scheduling exactly *one* exact alarm for the next upcoming event.
  - `DebtReminderReceiver`: Triggers on alarm wake‑up, sends standard high‑priority notification banners linked back to the dashboard, and programmatically schedules the next alarm.
  - `BootReceiver`: Automatically restores pending alarms after device reboots.
  - `Self-Healing Repository Hooks`: Hooks rescheduling checks into all database CRUD actions in `DebtRepository`, keeping alerts aligned with data.

### 4. 📂 Permissionless Auto-Backup & Data Import (CSV)
- **Settings Toggle Prefs**: Created **Backup / Auto save** and **Import / Restore** cards in `activity_settings.xml`.
- **Conditional Auto-Backup Observer**: Configured `DebtRepository` to read `pref_auto_backup_enabled` on every save, delete, or payment update, exporting active datasets dynamically only when enabled.
- **Initial Sync Force Export**: Toggling the switch to **ON** immediately performs a full export to external downloads, giving users immediate feedback.
- **Permissionless Modern MediaStore API**: Safely writes files to the user's `Download/Debt-Tracker` folder without needing high-risk external storage permissions.
- **Robust CSV Data Import & Overwrite**:
  - Built `importCsv` in `CsvBackupManager.kt` using a customized field splitter. It parses quoted cells, handles escaped delimiters, separates Debts from Payments, and restores entity IDs.
  - Integrates `MaterialAlertDialogBuilder` warning dialogues to confirm data overwrites.
  - Executes table purging and insertions sequentially using high-performance background coroutines, and programmatically reschedules alarms on completion to guarantee notification integrity.

### 5. 🎨 Clinical Minimalist Design & Theme Overlay Fixes
- **Unified Toolbar & App Backgrounds**: Blended custom Toolbars and Tab Layout containers with the unified app background (`#F6F8FA`) instead of standard white (`#FFFFFF`), ensuring complete app continuity.
- **Forced Theme Overlay Protection**: To resolve OEM dark mode force‑inversions (which generated unreadable dark text on dark grey windows), we implemented robust light-surface overlays:
  - Custom dialog style `ThemeOverlay_DebtTracker_MaterialAlertDialog` globally forces a light background (`#FFFFFF`) with contrasting dark text.
  - WheelPickers in `WheelPickerDialog` are programmatically wrapped in a solid white card view and the outer window is set to transparent (`dialog.window?.setBackgroundDrawableResource(android.R.color.transparent)`).
  - Exposed dropdown menu popups are locked using `android:popupBackground="@color/dt_surface"` and item selections use [item_dropdown.xml](file:///e:/Github/Debt-Tracker/app/src/main/res/layout/item_dropdown.xml) to enforce black text on white under all conditions.
- **Typography and Border Polish**: Enlarged wheel picker fonts to `26sp` inside custom theme overlays. Refactored the info button (`ibPreviewInfo`) to inherit from standard Material ImageButtons with unified paddings, borders, and touch-ripples.

## 🛠️ Refactoring & Compilation Stability

1. **Jetpack Room Upgrade & KSP Migration**:
   - Replaced legacy, slow `kapt` annotation compilers with modern, high-performance **Kotlin Symbol Processing** (`ksp`).
   - Upgraded Room database from `2.6.1` to `2.8.4` to fix Kotlin metadata version compiler mismatches with AGP 9.1.1.
   - Cleared deprecated AGP flags in `gradle.properties` and resolved standard built-in compiler options.
2. **LiveData Transformation Syntax**:
   - Replaced deprecated `Transformations.map` in `DebtController.kt` with modern Kotlin DSL lifecycle extensions (`liveData.map { ... }`).
3. **Android 14 Activity Transitions**:
   - Replaced deprecated `overridePendingTransition` with a custom API-aware extension `overrideSlideTransition(isEntering)` in `TransitionUtils.kt` to ensure compliance with API 34+ guidelines.
4. **RecyclerView ViewHolder Position**:
   - Swapped deprecated `adapterPosition` with modern `bindingAdapterPosition` in `DebtListActivity.kt`.
5. **XML and String Pruning**:
   - Safely removed all unused index references from `strings.xml` following the removal of the redundant notification Interval settings.
