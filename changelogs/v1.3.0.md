# 📜 Change Log — Version v1.3.0

**Release Date:** May 19, 2026  
**System Architecture:** Multi-Language Strings Resource Mapping, Real-Time Currency Watcher on Obligation Principal, Custom Dialog Theme Overrides, and Unified Toolbar Edit Access Flow.

This release introduces complete **Multi-Language Expansion & Localization Alignment** across all 8 language folders, wires the real-time **CurrencyTextWatcher Auto-Formatting** to the debt principal (Nợ gốc) input form, resolves critical editing and serialization data saving bugs, and integrates a premium **Toolbar Edit Option** and highly consistent **Material 3 Dialog Theme Overrides** to achieve absolute visual and structural excellence.

---

## 🚀 New Features & Enhancements

### 1. 🌐 Complete Multi-Language Detail & Card Status Localization
- **8 Locale Mappings**: Added fully translated, comprehensive resource keys (`next_due_label`, `remaining_months_label`, `total_paid_label`, `debt_principal_label`, `loan_type_label`, `status_overdue`, `status_upcoming`, `status_no_due`, `status_expired`, `months_format`) across all 8 supported language resource subfolders (Default/EN, UK English, Vietnamese, Japanese, Korean, Simplified Chinese, and Traditional Chinese).
- **Detail Layout Localization**: Fully localized all hardcoded labels inside `activity_debt_detail.xml` using dynamic string resources.
- **Card Adapter Status Localization**: Localized all status tags ("Trễ hạn", "Sắp đến", "Không có kỳ hạn", "Hết hạn") in both `UpcomingAdapter` (in `MainActivity.kt`) and `DebtListAdapter` (in `DebtListActivity.kt`).
- **Months Format Suffix Refactoring**: Completely eliminated the brittle Vietnamese-specific `.replace("Số tháng còn lại: ", "")` string truncation hack inside `DebtDetailActivity.kt`. Introduced a cleanly parameterized string resource `months_format` to format and render months seamlessly according to different localized suffix rules (e.g. "12 months" in English, "12 tháng" in Vietnamese, JPY, KRW, and Chinese suffix rules).
- **Fallback Status Mapping**: Localized fallback dates inside `DebtDetailActivity.kt` using the standard `R.string.status_no_due` resources.

### 2. ⌨️ Real-Time Currency Watcher Auto-Formatting on Principal (Nợ gốc)
- **Live Formwatcher Binding**: Attached our custom, real-time dynamic `CurrencyTextWatcher` to `editPrincipal` (Nợ gốc) inside `CreateEditDebtActivity.kt`.
- **Digit Scaling Calculations**: Programmatic principal initialization in edit mode and automatic amount-to-duration multiplication in `calculateAndSetPrincipal()` format raw double entries to normalized digit sequences, making the watch-tracked text box format dynamically in real-time.
- **Double Variable Parameterization**: Bound a companion private variable `rawPrincipal: Double` to track and maintain accurate unformatted numeric entries directly inside `CreateEditDebtActivity.kt`.

### 3. 🛠️ Critical Edit Form Bug Fixes & Unified Edit Option Flow
- **Formatted-String Parsing Fix**: Solved a major database saving bug where parsing formatted principal strings (`10.000.000` or `$10,000.00`) directly with `toDoubleOrNull()` failed and returned `0.0`, resulting in incorrect database records. Integrated real-time companion `rawPrincipal` double tracking.
- **Form Population Fix**: Populated `editCreditor` and `editContract` inside `loadDebtDetails` (they were previously completely omitted, causing empty edit forms on load).
- **Edit Icon Toolbar**: Created a standard Material Design vector asset `ic_edit.xml`, created the menu layout `menu_debt_detail.xml`, and registered menu handlers in `DebtDetailActivity.kt` to allow users to trigger edits seamlessly with a beautiful icon.
- **Crash-Proof Date Parsing**: Wrapped all standard month and date parsing procedures (`remainingMonths`, `monthsBetweenInclusive`, `isActiveForMonth`, `computeNextDueDate`) inside `DateUtils.kt` and `CreateEditDebtActivity.kt` in try-catch safety nets. This ensures that any missing, unparsed, or malformed date fields in the database will never crash or break the detail screen, dashboard, or creation/edit forms.
- **Robust Key-Value Row Layout**: Replaced the faulty `GridLayout` in `activity_debt_detail.xml` (which had omitted width dimensions on columns, pushing them off-screen or rendering them blank) with a bulletproof nested horizontal-row vertical layout hierarchy. This ensures all values (Monthly amount, Next Due Date, Remaining Months, Total Paid) are perfectly right-aligned, visible, and fully responsive across all device screen sizes.

### 4. 🎨 Unified Dialog Theme Styling & Android OS Override Protection
- **Dialog Theme Overlay Override**: Enforced consistent app styling (Indigo-accented elements, crisp rounded white cards, accessibility-compliant typography sizes) by creating customized overlays for material dialog alerts in both `values/themes.xml` and `values-night/themes.xml` using `ThemeOverlay_DebtTracker_MaterialAlertDialog`.
- **Dialog Buttons styling**: Created `Widget_DebtTracker_DialogButton` wrapping active text buttons with solid `@color/dt_primary` branding and dynamic container highlights to bypass Android dynamic theme or grey system overrides.
- **Explicit Dialog Background Enforcements**: Explicitly declared solid `dt_surface` backgrounds on dialog windows, ensuring dynamic dark mode overrides by Android OS never compromise legibility.

### 5. 🔔 Auto-Advancing Due Dates & Prepayment Notification Bypassing
- **Oldest Unpaid Date Targeting**: Refactored `DateUtils.computeNextDueDate()` to start tracking and scanning from the oldest unpaid installment month (calculated dynamically by matching `debt.totalPaid` against monthly installments).
- **Auto-Advancing Due Dates**: Due date calculations now automatically shift ahead to reflect exact future unpaid intervals if a user pays in advance or prepays (e.g. paying 3 months ahead will instantly shift their next visible due date 3 months into the future on all display cards and detail screens).
- **Notification Prepayment Bypass**: Refactored `DebtReminderReceiver.kt` to dynamically calculate expected accumulated installments up to the current calendar month. If the accumulated payments (or current month logs) fully satisfy the obligation threshold, the alarm receiver automatically skips firing push notifications for that debt.
- **Dashboard Progress Alignment**: Refactored `DashboardController.kt` to count prepaid debts as fully paid for the current month, ensuring dashboard circular progress indicators, paid-counter status fields, and timelines align seamlessly with prepaid records.
- **Dynamic Remaining Due**: Enforced the dashboard metric `totalDueThisMonth` to represent the **remaining unpaid amount due** instead of the overall monthly total. Once a payment is recorded or prepaid for a debt, the "Total due this month" automatically decreases by that debt's monthly installment amount (implementing: `Due amount = Total - Debts paid`). E.g. when all active debts for the month are paid, the dashboard total due instantly goes to 0.

### 6. 📅 Customizable Date Formatting Settings
- **Exposed Dropdown Date Selector**: Created a premium outlined exposed dropdown selection card inside the Settings screen (`activity_settings.xml`) utilizing material `TextInputLayout` and `AutoCompleteTextView` containing 3 standard choice options:
  1. `DD/MM/YYYY` (e.g. 19/05/2026) - default standard layout
  2. `MM/DD/YYYY` (e.g. 05/19/2026) - US standard layout
  3. `YYYY-MM-DD` (e.g. 2026-05-19) - ISO international standard layout
- **Unified Settings Controller Integration**: Integrated date preferences inside `SettingsActivity.kt` to load the current preference on launch and save the preferred pattern securely to `debt_tracker_settings` SharedPreferences upon choice selection.
- **Dynamic Localized Formatting**: Refactored `DateUtils.formatDay` (along with all its associated calls inside `MainActivity.kt`, `DebtListActivity.kt`, and `DebtDetailActivity.kt`) to accept a `context` parameter and format all visible calendar dates dynamically according to the user's selected preference.
- **Full Multi-Language Translation support**: Added date format translation labels across all 8 language subdirectories (`strings.xml` resource bundles).

### 7. ↕️ Collapsible Settings Accordion Sections
- **Interactive Section Headers**: Replaced all static setting segment titles (General, Notifications, Backup, Import/Restore) inside the settings list with high-end interactive row containers (`headerGeneral`, `headerNotifications`, `headerBackup`, `headerImport`) containing custom styled `ImageView` chevrons (`ic_chevron_down.xml`) on the right side.
- **Collapsed by Default Layout**: Wrapped setting options into four separate logical section containers initialized as hidden (`View.GONE`) by default, yielding a clean, completely uncluttered settings dashboard at first glance.
- **Fluid Chevron Rotation Animations**: Implemented an automated click-toggle listener inside `SettingsActivity.kt` which animates the drop-down chevrons smoothly between `0` degrees (facing down) and `180` degrees (facing up) in absolute harmony with the toggle visibility changes.
